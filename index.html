03 / Implementácia

Kód a algoritmy

ExpectiMax + zdieľaná eval funkcia s monotonicitou, smoothness, empty cells a corner bonusom.

evaluate() — eval funkcia heuristiky a ExpectiMaxu
# Zdieľaná eval funkcia pre oba solvery def evaluate(grid): mono = eval_monotonicity(grid) smooth = eval_smoothness(grid) empty = eval_empty(grid) corner = eval_corner(grid) return ( 1.0 * mono + 0.1 * smooth + 2.7 * empty + 1.0 * corner ) # ExpectiMax — max node def expectimax(grid, depth, is_max): if depth == 0: return evaluate(grid) if is_max: best = -inf for move in ['L','R','U','D']: ng = apply_move(grid, move) if changed(ng, grid): best = max(best, expectimax(ng,depth-1,False)) return best else: # chance node val, empty_cells = 0, get_empty(grid) for cell in empty_cells: for tile,p in [(2,0.9),(4,0.1)]: ng[cell] = tile val += p * expectimax( ng,depth-1,True) / len(empty_cells) return val

07 / Výzvy a objavenia

Problémy z praxe

Seed contamination bug, overfit corner váhy, nbclient asyncio — reálne debuggovanie AI projektu.

Hlavný bug
Seed Contamination
Nová greedy heuristika (~490 ťahov/hra) vs. stará (~200) konzumovala iný počet náhodných čísel — ExpectiMax dostal iné sekvencie spawnovania. Oprava: np.random.seed(42) pred každým solverom.
Overfit váha
CORNER_W = 2.0 Hurt
Zdvojnásobenie corner bonusu (1.0 → 2.0) znížilo výhry z 24 na 20/30. Príliš silný corner tlak obmedzil flexibilitu v mid-game. Vrátili sme sa na 1.0.
Runtime bug
nbclient asyncio uviazol
Jupyter nbclient zasekol na 0% CPU bez chybovej správy. Riešenie: exportovať kód z notebooku do standalone Python skriptu (/tmp/notebook_standalone.py) a spustiť priamo.
Split eval test
Oddelené eval funkcie
Beh č.5 testoval rozdielne eval funkcie pre Heuristic a ExpectiMax (corner_bl len pre EM). Výsledok: ExpectiMax drop na 17/30. Zdieľaná eval funkcia je lepšia — consistent heuristika.
Príliš pomalé
CRITICAL_THRESHOLD = 4
Threshold=4 prázdnych políčok pre depth=4 spomalil hru 6× bez merateľného zlepšenia. Threshold=2 je správna voľba — depth-4 iba v skutočnom endgame.
Štatistika
Malý vzorec klame
5-game tuning ukázal bl+depth4 ako "dokonalý" (5/5 wins). 30-game test: len 1 extra výhra oproti baseline. Na spoľahlivú štatistiku treba n≥70 hier (σ=1513).

expectimax.py
def expectimax(grid, depth, is_max): if depth == 0: return evaluate(grid) if is_max: # MAX-node best = -inf for m in ['L','R','U','D']: ng = apply_move(grid, m) if changed(ng): best = max(best, expectimax(ng,depth-1,False)) return best else: # CHANCE-node val, cells = 0, empty(grid) for c in cells: for t,p in [(2,.9),(4,.1)]: ng[c] = t val += p * expectimax(ng,depth-1,True) / len(cells) return val

if empty_count(grid) <= 2: depth = 4 # endgame else: depth = 3 # štandard

05 / Iteratívny vývoj

5 behov, 1 kritická chyba, 8 konfigurácií

Reálny výskumný cyklus — vrátane identifikácie seed contamination bugu ktorý zmenil výsledky o 17 %.

Beh č.1 · pevná priorita smerov
Baseline implementácia
Heuristika s pevnou prioritou down>left>up>right. Zdieľaný seed pre všetkých solverov.
Heuristic avg 2 193 ExpectiMax 24/30 (80%) seed artefakt!
Beh č.2 · greedy 1-ply eval
Heuristika vylepšená, ExpectiMax klesol
Greedy eval výrazne zlepšil heuristiku. Ale ExpectiMax klesol na 17/30 — seed contamination!
Heuristic +229% ExpectiMax 17/30 (57%)
Beh č.3 · seed reset oprava
Férové porovnanie po oprave
Seed reset pred každým solverom — každý dostane rovnaké hry. Reálny výkon ExpectiMaxu odhalený.
ExpectiMax 19/30 (63%) reálny výkon
Beh č.4–5 · systematický tuning
8 konfigurácií × 5 hier
bl+depth4 bol najlepší v 5-hrovom teste (100%). V 30-hrovom teste: 20/30 — marginálne zlepšenie, štatisticky nepreukazné.
+1 výhra oproti beh3 σ=1513, n≥70 hier
Seed contamination — prečo 80 % ≠ reálny výkon
Beh č.1 stará heuristika ~200 ťahov → ExpectiMax od pozície ~9 570 24/30 ★
Beh č.2 greedy heuristika ~490 ťahov → ExpectiMax od pozície ~18 300 17/30 ↓
Beh č.3 seed reset pred každým → každý od pozície 0 19/30 ✓
